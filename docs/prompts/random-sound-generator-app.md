# Промпт для AI-агрегатора: приложение генерации случайных звуков

Скопируй текст ниже и отправь AI-агрегатору/конструктору приложений.

```text
Создай полностью готовое к запуску веб-приложение "Random Sound Lab" для генерации случайных звуков прямо в браузере.

Цель приложения:
Пользователь нажимает кнопку и получает новый случайный звук. Звук должен генерироваться программно через Web Audio API, без обязательной загрузки внешних аудиофайлов. Приложение должно работать локально сразу после установки зависимостей и запуска команды разработки.

Технологический стек:
- Next.js 14+ с App Router.
- React + TypeScript.
- CSS Modules, Tailwind CSS или обычный CSS — выбери простой и надежный вариант.
- Используй Web Audio API для синтеза звука в браузере.
- Не используй платные API и внешние сервисы.
- Не требуй backend, базу данных или авторизацию.

Основные функции:
1. Кнопка "Сгенерировать звук".
   - При нажатии создается случайный звук длительностью примерно 0.2–3 секунды.
   - Звук должен проигрываться сразу после генерации.

2. Случайные параметры звука:
   - Тип осциллятора: sine, square, sawtooth, triangle.
   - Частота: случайное значение примерно от 80 до 2000 Hz.
   - Длительность: случайное значение от 0.2 до 3 секунд.
   - Громкость: безопасное значение, чтобы звук не был слишком громким.
   - Envelope/огибающая: плавная атака и затухание, чтобы не было резких щелчков.
   - Дополнительно: случайный фильтр lowpass/highpass/bandpass, легкое vibrato или шумовой слой.

3. Режимы генерации:
   - "Простой тон" — один осциллятор.
   - "Шум" — белый/розовый шум через AudioBuffer.
   - "Аркада" — короткий ретро-звук с изменением частоты.
   - "Атмосфера" — более длинный мягкий звук с фильтром.
   - "Случайный режим" — приложение само выбирает один из режимов.

4. Управление:
   - Кнопка "Стоп" для немедленной остановки текущего звука.
   - Ползунок общей громкости.
   - Переключатель "Автогенерация" — если включен, новый звук создается каждые 1–5 секунд.
   - Кнопка "Случайные настройки".

5. История генераций:
   - Покажи список последних 10 звуков.
   - Для каждого элемента отобрази режим, частоту, длительность, тип волны и время создания.
   - Добавь кнопку "Повторить" для повторного проигрывания звука с теми же параметрами.

6. Экспорт:
   - Если возможно без сильного усложнения, добавь кнопку "Скачать WAV" для последнего сгенерированного звука.
   - Для экспорта используй OfflineAudioContext и сформируй WAV-файл на клиенте.
   - Если экспорт реализовать сложно, оставь аккуратную заглушку с понятным TODO-комментарием в коде, но основной генератор обязан работать.

UI/UX:
- Визуал простой, современный и адаптивный.
- Центрированный интерфейс: заголовок, короткое описание, панель управления, текущие параметры, история.
- Добавь темный фон или светлую минималистичную тему, мягкие карточки, понятные кнопки.
- На мобильных устройствах интерфейс должен оставаться удобным.
- Добавь предупреждение/подсказку: "Начинайте с низкой громкости".

Требования к качеству:
- Код должен быть чистым, типизированным и разделенным на понятные части.
- Не создавай утечки AudioContext: корректно останавливай и отключай узлы.
- Обработай ограничения браузера: AudioContext должен запускаться после пользовательского действия.
- Не допускай слишком громких или неприятно резких звуков: используй gain envelope и ограничение громкости.
- Не должно быть ошибок TypeScript, ESLint и runtime-ошибок в консоли.

Структура проекта:
- Создай все необходимые файлы проекта.
- Добавь README.md с описанием, командами запуска и кратким объяснением функций.
- Минимальные команды должны быть:
  - npm install
  - npm run dev
  - npm run build

Ожидаемый результат:
- Полностью готовое приложение, которое можно запустить локально.
- После запуска пользователь открывает страницу в браузере, нажимает "Сгенерировать звук" и сразу слышит случайный звук.
- Приложение не зависит от внешних аудиофайлов, серверов или платных сервисов.
```
